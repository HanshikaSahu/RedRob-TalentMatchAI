# TalentMatchAI — Intelligent Candidate Ranking System

## Overview
TalentMatchAI is an intelligent candidate discovery and ranking system developed for the India.Runs Data & AI Challenge by Redrob AI. The system evaluates and ranks 100,000 candidate profiles against a Senior AI Engineer role by combining semantic understanding of career histories with behavioral, experience, and verification signals.

Rather than relying solely on keyword matching, the solution attempts to replicate how an experienced recruiter evaluates candidate fit by considering real work experience, platform activity, assessment performance, availability, and domain relevance.

## Approach

### Stage 1 — Hard Disqualification Filter
Before any ML runs, candidates are eliminated if they:
- Have worked exclusively at consulting firms (TCS, Infosys, Wipro, Accenture, Cognizant, Capgemini)
- Have been inactive on the platform for 6+ months

This removes ~30% of candidates instantly.

### Stage 2 — Semantic Scoring
Uses sentence-transformers (all-MiniLM-L6-v2) to embed the job description and each candidate's career history into vector space. Cosine similarity measures how close a candidate's actual work experience is to what the role needs. Career history descriptions are scored — not just the skills list — because the JD explicitly warned that keyword-stuffed skill sections are a trap.

### Stage 3 — Multi-Signal Composite Score

| Signal | Weight | What it measures |
|--------|--------|-----------------|
| Semantic match | 35% | Career descriptions vs JD via embeddings |
| Experience quality | 25% | Product company history, years in applied ML |
| Behavioral availability | 20% | Active, open to work, response rate, notice period |
| Verified signals | 12% | Assessment scores, GitHub activity, verified contacts |
| Nice-to-haves | 8% | LoRA, open-source, HR-tech domain experience |

### Penalty Multipliers
- Title penalty (0.25x) for irrelevant titles
- Job hopper penalty (0.5-0.7x) for 3+ roles under 12 months
- Semantic threshold: candidates below 0.28 cosine similarity scored as 0

## Results
- 100,000 candidates processed
- ~29,000 hard disqualified upfront
- Top 100 all have directly relevant ML/AI titles and 5-9 years experience
- Rank 1: AI Engineer, 6.9 years, score 0.7184

## Tech Stack
- Python 3.13
- sentence-transformers (all-MiniLM-L6-v2)
- scikit-learn
- numpy
- CPU only, no GPU required

## Pipeline & Compute Strategy
The system is split into two stages to satisfy the 5-minute / 16GB / CPU-only ranking constraint:

**precompute.py** (one-time, ~2 hours)
Streams candidates.jsonl line by line, applies the hard disqualification filter, and encodes the remaining ~70,000 survivor profiles into embeddings using sentence-transformers. Saves three artifacts to disk: embeddings.npy, survivor_ids.json, and survivor_profiles.json. This step is allowed to exceed the 5-minute window per the submission spec, since it is offline pre-computation.

**rank.py** (the actual ranking step, ~35 seconds)
Loads the pre-computed embeddings and profiles directly from disk — it never re-reads candidates.jsonl. Computes cosine similarity against the job description, applies the multi-signal scoring function, and writes the final ranked submission.csv. Well within the 5-minute / 16GB / CPU-only constraint.

## Reproduce Submission

Install requirements:
```
pip install -r requirements.txt
```

Step 1 — One-time pre-computation (~2 hours, generates embeddings.npy, survivor_ids.json, survivor_profiles.json):
```
python precompute.py
```

Step 2 — Generate ranking (~35 seconds):
```
python rank.py --candidates ./candidates.jsonl --out ./submission.csv
```

Step 3 — Validate:
```
python validate_submission.py submission.csv
```
